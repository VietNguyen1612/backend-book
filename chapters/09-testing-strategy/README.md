# Chapter 9: Testing Strategy

A comprehensive testing strategy is the foundation of reliable software. This chapter covers how to structure your tests across the testing pyramid and the practices that make your test suite trustworthy and maintainable.

## Table of Contents

1. [9.1 Testing Pyramid](testing-pyramid.md) -- Unit tests (AAA pattern, parametrize, edge cases, property-based testing with Hypothesis), integration tests (testcontainers, API tests, database tests, test doubles), and performance testing (Locust, load/stress/soak/spike tests, benchmarking).

2. [9.2 Testing Practices](testing-practices.md) -- F.I.R.S.T. principles, test coverage, test data with factory_boy and Faker, mutation testing with mutmut, and contract testing with Pact.
